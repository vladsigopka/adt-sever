# АДТ Север — верстка

Статическая верстка (HTML / SCSS / vanilla JS) под последующее натягивание на **1С-Битрикс**.

## Стек

- **Gulp 5** — сборка (`gulpfile.mjs`, ESM)
- **gulp-file-include** — сборка HTML из частей (`@@include`); на выходе чистый HTML
- **SCSS + БЭМ** — стили (dart-sass, `@use`)
- **Vanilla JS (ES6+)** — без jQuery; модули из `js/components/*` склеиваются в `main.js`
- **browser-sync** — локальный сервер с авто-перезагрузкой

## Команды

```bash
npm install      # один раз
npm run dev      # сборка + dev-сервер (http://localhost:3000), watch + live-reload
npm run build    # прод-сборка в dist/
npm run clean    # очистить dist/
```

Отдаётся на натягивание папка **`dist/`** (чистый HTML/CSS/JS, без следов Gulp).

## Плавный скролл (Lenis)

Инерционный скролл всего сайта — библиотека **Lenis**, завендорена локально (`src/js/vendor/lenis.min.js`, UMD → глобальный `Lenis`), склеивается в `main.js`. Инициализация — `js/components/smooth-scroll.js`. Мягкость регулируется там же: `duration` (длина доката, сейчас 1.8 — больше = мягче) + `wheelMultiplier` (отклик на колесо, 0.9 — меньше = нежнее).

- CSS Lenis — `scss/base/_lenis.scss` (служебные классы `.lenis-smooth`, `.lenis-stopped` и т.п.).
- Отключается при `prefers-reduced-motion: reduce` (доступность).
- Якорные ссылки `a[href^="#"]` перехватываются и скроллятся через `lenis.scrollTo`.
- При открытии бургер-меню скролл фона ставится на паузу: `window.lenis.stop()` / `.start()` (см. `mobile-menu.js`).
- Тач-скролл не трогаем (нативный) — `syncTouch` по умолчанию выключен.
- Если на странице появится контейнер со своим скроллом (модалка, таблица) — повесить на него `data-lenis-prevent`.
- **Битрикс:** Lenis чисто клиентский, не зависит от сборки и бэкенда — после натягивания работает как есть (файл лежит локально в `dist/js/main.js`).

### Кастомный скроллбар
Тонкий кастомный скроллбар справа — `js/components/custom-scrollbar.js` + `scss/blocks/_custom-scroll.scss`. Рисует положение скролла (работает в паре с Lenis: Lenis скроллит, бар рисует). Появляется при скролле/наведении на правую кромку, прячется через 0.5с. Перетаскивание thumb двигает страницу через `lenis.scrollTo(..., { immediate: true })`.
- Нативный скроллбар скрыт классом `html.custom-scroll-ready` (его вешает JS).
- Только для мыши: при `pointer: coarse` (тач) не инициализируется и нативный остаётся.
- Прячется при открытом бургер-меню.

## Анимации появления (reveal)

Плавное появление при скролле — атрибут `data-reveal` на элементе (`js/components/reveal.js` + `scss/base/_reveal.scss`). Элемент стартует прозрачным со сдвигом вниз, при попадании в вьюпорт (IntersectionObserver) получает класс `.is-revealed`. Опционально стаггер — `data-reveal-delay="120"` (мс).
- Скрытое состояние — только под `html.js-ready`: если JS не сработал, контент виден.
- Отключается при `prefers-reduced-motion` (и в CSS, и в JS).
- Применено на странице акций (`akcii.html`): заголовок + карточки `.promo-card`.
- Ховер карточек (десктоп): зум картинки `scale(1.05)` + мягкая тень (`_promo-card.scss`).

## Хедер — закреплён + скрытие при скролле

`.header` — `position: sticky; top: 0; z-index: 90` (ниже бургер-меню 100, модалок 200, кастомного скроллбара 9998+; фон непрозрачный `$color-page`). Поведение «hide-on-scroll-down»: `js/components/header-scroll.js` по `scroll` окна (Lenis скроллит реальное окно → события приходят, `rAF`-троттлинг) переключает класс `.header--hidden` (`transform: translateY(-100%)`, transition 0.35s):
- скролл ВНИЗ (ниже высоты хедера) → хедер уезжает вверх (прячется);
- скролл ВВЕРХ → возвращается; у самого верха (`y ≤ высота хедера`) — всегда показан.
- Порог 8px против дребезга; транзишен отключается при `prefers-reduced-motion`.
- **Связка со sticky-сайдбаром ЛК:** `.account__sidebar` прилипает на `top: rem(96px)` (= высота хедера 72 + 24 зазор), чтобы не заходить под хедер.

## Ключевые соглашения (ВАЖНО)

### 1. Без комментариев в коде
В исходниках (HTML/SCSS/JS, gulpfile) комментариев нет. Документация — только здесь, в README.

### 2. Цвет текста через opacity
Иерархия текста = **чёрный + прозрачность**, не серые цвета:
- основной текст — `opacity: 1`; описание — `opacity: 0.8`; дата/мета — `opacity: 0.5`.
- Если в строке смешаны приглушённый текст и яркие ссылки/акценты — `opacity` на родителе **обрежет детей**. В таких случаях: `color: rgba($color-black, .5)` на тексте + полный цвет на ссылке (см. `auth__agreement`, `auth__switch`).

### 3. Резиновость — модель «резиновый корень»
- На `html` стоит `clamp()` на `font-size` (14px→16px между 1024 и 1920px) — см. `base/_base.scss`.
- Поэтому **всё в `rem()`** автоматически масштабируется. Функция `rem($px)` — в `base/_functions.scss`.
- Функции `fluid()` **нет** (удалена) — не использовать, иначе двойное масштабирование.
- В `px` оставляем только то, что НЕ должно тянуться: рамочный гуттер 24px, `1px`-бордеры.

### 4. Один брейкпоинт: десктоп ≥ 1024, мобилка < 1024
- Переменная `$bp-mobile: 1024px`, миксин `@include upto($bp-mobile) { ... }` (он вычитает 0.2px, чтобы 1024 относился к десктопу).

### 5. Рамка страницы (гуттер)
- `body` (кроме `.body--auth`): `padding-inline: 24px` + `padding-bottom: 24px` (футер инсетный — отступ 24px по бокам и снизу).
- Узкие колонки: `.container--narrow` (1040px), `.container--content` (для детальных/контентных страниц).

### 6. Иконки — инлайн-спрайт
- Все иконки в `partials/sprite.html` как `<symbol id="...">` с `currentColor`.
- Использование: `<svg><use href="#icon-name"></use></svg>`. Цвет — из CSS (`color`).
- Исходники — в `src/icons/`. Спрайт подключается один раз после `<body>` на каждой странице.
- Есть: `logo`, `icon-comparison`, `icon-heart`, `icon-bag`, `icon-user`, `icon-minus`, `icon-document`, `icon-chevron`, `icon-eye`, `icon-arrow` (стрелка слайдера — вправо, prev поворачивается на 180°), `icon-search` (лупа), `icon-video` (play-кружок — для бейджа «Видеообзор» и кастомной кнопки плеера), `icon-arrow-long` (длинная стрелка-вправо — для стрелок слайдера «В наличии», prev rotate 180°).

### 7. Картинки — только через `<picture>`
Партиал `partials/picture.html` (webp + растровый фолбэк). Пример:
```html
@@include('partials/picture.html', { "webp": "img/x.webp", "img": "img/x.png", "alt": "...", "class": "block__image" })
```

**Оптимизация при сборке (sharp).** В `src/img/` кладётся ОДИН исходник (`.png`/`.jpg`) — задача `images` в `gulpfile.mjs` сама генерит в `dist/img/`:
- оптимизированный `.webp` (`quality 80`) — основной формат для `<source>`;
- сжатый фолбэк того же имени (PNG — `compressionLevel 9` + палитра; JPG — `quality 82`, mozjpeg) для `<img src>`;
- даунскейл до `IMG_MAX_WIDTH` (1920px) — большие фото уменьшаются без видимой потери качества.

Поэтому webp вручную делать НЕ нужно (раньше делали). SVG/прочее копируются как есть. Исходные `.webp` в `src/img/` (если есть png-двойник) не копируются — пересоздаются из исходника. Пример выигрыша: фон слайда 1.9 МБ PNG → 57 КБ webp.

### 8. Шрифт
**RF Dewi Expanded** (широкое начертание), вес 400/600/700. `@font-face` — в `base/_fonts.scss`, `$font-base` — в переменных. Веса 500 в шрифте нет → для акцента ставить 600.

### 9. Кнопки
`.btn` — базовая высота 54px (через `height`), размеры `--sm`/`--lg`, цвета `--primary`/`--outline`, ширина `--block`. Анимация: hover — лёгкий подъём + мягкая тень; active — «вдавливание».

### 10. Поля формы (важный паттерн)
Кастомный плейсхолдер (нативный не может быть двухцветным — серый текст + чёрная «*»):
```html
<input class="field__input" placeholder=" " aria-label="Введите логин" required />
<span class="field__label">Введите логин<span class="field__star">*</span></span>
```
- `placeholder=" "` (пробел) + `aria-label` (доступность).
- Скрытие лейбла при вводе — через JS-класс `.field--filled` (`field-label.js` слушает `input/change/load`). **Не использовать `:placeholder-shown`** (см. грабли).
- Автозаполнение: лейбл скрывается по `:-webkit-autofill`, голубой фон убран box-shadow-хаком.

## Главная — hero + конфигуратор (index.html)

Первый экран — слайдер `.hero` (скруглённый блок на всю ширину контейнера) с оверлеями поверх дорожки слайдов:
- **Слайды** `.hero__slide` — фон `.hero__media` (`<picture>`, `object-fit: cover`, `z-index: 0`) + затемняющий градиент-оверлей `.hero__slide::after` (`z-index: 1`, слева темнее — чтобы белый заголовок читался); бейджи `.hero__badge` (`--light`/`--dark`) слева-сверху, контент (заголовок/текст/кнопка «Смотреть») по центру слева — всё `z-index: 2`. Каждый слайд несёт свои бейджи и заголовок. Сейчас на всех слайдах одно фото `img/hero/slide-1` (заменить на разные). У `.hero__viewport` — `cursor: grab` (`:active` → grabbing), на кнопках/ссылках `cursor:pointer`, на `img` `-webkit-user-drag:none` (свайп мышью без перетаскивания картинки).
- **Кнопка «Online — покупка»** `.hero__online` — белая пилюля (чёрный текст) по центру сверху; «волны расходятся от самой кнопки РАВНОМЕРНО во все стороны»: два span `.hero__online-ripple` (`position:absolute; inset:0; border-radius:999px` (полный pill); `z-index:-1` → прячутся за фоном пилюли, видны только выходящие за края белые кольца). Анимация `hero-online-wave` — **только `transform` + `opacity`** (композитятся на GPU, фон НЕ перерисовывается → плавно даже поверх большого фото), `will-change: transform, opacity`, со сдвигом фазы (2-е кольцо `animation-delay`). Равномерность (одинаковый прирост в px со всех сторон при разном W/H пилюли) — через **раздельные scaleX/scaleY в пропорции сторон** (`scale(1.16, 1.69)` при ≈210×49: ~+16.8px по обеим осям). Перебор приёмов (грабли — НЕ повторять): `transform: scale(uniform)` тянет шире по горизонтали; `inset`/`outline-offset` равномерны, НО это layout/paint-свойства → дёрганая анимация поверх тяжёлой фото-картинки. Итог — разнесённый scaleX/scaleY. Плюс акцентная точка-маркер `.hero__online-dot`.
- **Стрелки** `.hero__nav` `.hero__arrow` — слева-снизу, КВАДРАТНЫЕ (radius 16, `icon-arrow`, prev повёрнута на 180°). Активная: белый фон, чёрная стрелка, без рамки (border `transparent` для стабильного размера). Неактивная (`:disabled` на крайних слайдах): прозрачный фон + `border: 1px solid rgba(255,255,255,.15)` (#FFFFFF26) + стрелка `rgba(255,255,255,.4)`. Для этого слайдер на главной — `data-slider-loop="false"` (без автоплея): на первом слайде гаснет prev, на последнем — next.
- **Обёртка `.hero-block`** (`position:relative; margin-top:24px`) — содержит `.hero` + `.configurator`. `margin-top:24px` ФИКС (рамочный гуттер от хедера до слайдера, как `body padding` — не `rem`). Конфигуратор вынесен ИЗ `.hero` сюда, чтобы на мобилке стать отдельным блоком ПОД слайдером (вне `overflow:hidden` hero); `.hero-block` — контекст для абсолюта карточки на десктопе.
- **Конфигуратор** `.configurator` — на десктопе белая плавающая карточка над фото на ВСЮ высоту слайда: `position:absolute; top/right/bottom:32` (инсет 32), `width:440`, radius 20, тень. Внутри flex-column: title сверху, `__form{flex:1}` → `__fields{flex:1; justify-content:space-between}` — 9 селектов РАСПРЕДЕЛЯЮТСЯ, заполняя высоту без пустот (зазор ~18px), снизу «Дополнительные поля» + чёрная «Показать N автомобилей» (`icon-search`). Высота слайда (и всего hero) — `min-height: rem(832px)` на `.hero__slide`. Чтобы 9 полей + 2 кнопки влезли в эту высоту с инсетом 32 БЕЗ переполнения (иначе кнопка «Показать» вылезала за слайдер), внутри конфигуратора селекты ужаты: `.configurator .select__trigger{height:44px}` (вместо дефолтных 54 — скоупом, глобальный селект не трогаем), `__fields` gap 10, title/кнопки чуть компактнее. На мобилке — `position:static` → отдельный блок-карточка под слайдером (flex/space-between без эффекта, обычный gap 16).
- **Чтобы текст слайда не залезал под карточку** при сужении: слайду задан `padding-right: rem(504px)` (резерв зоны карточки = 32 inset + 440 ширина + ~32 зазор) → текст переносится, не доходя до неё. Кнопка `.hero__online` центрируется НЕ по всему hero, а по «фотозоне» слева: `left: calc((100% - 504px) / 2)` — иначе на узких (≤1024) наезжала бы на карточку. На мобилке оба резерва сбрасываются (слайд `padding:24`, online `left:24`).

### Слайдер (переиспользуемый, `components/slider.js`)
Vanilla, без библиотек (на главной несколько слайдеров — Swiper не тянем, чтобы не плодить отдельный CSS и держать бандл лёгким; если позже понадобятся thumbnails/loop/тач-инерция — можно завендорить как Lenis).
- Разметка: контейнер `[data-slider]`, дорожка `[data-slider-track]` (прямые дети = слайды). Опционально: `[data-slider-prev]`/`[data-slider-next]`, `[data-slider-dots]` (точки генерятся в JS), `[data-slider-current]`/`[data-slider-total]` (счётчик).
- Атрибуты: `data-slider-autoplay="7000"` (мс; пауза при наведении; при `loop=false` — **пинг-понг**: дойдя до конца, едет назад, без резких перескоков), `data-slider-loop="false"` (выключить зацикливание → стрелки `disabled` на краях/конце прокрутки).
- Свайп мышью/тачем по всему блоку; элементы с `data-slider-no-swipe` свайп не перехватывают.
- **Шаг — по реальной позиции слайда** (`slides[i].offsetLeft`), с клампом до конца прокрутки (`scrollWidth - parent.clientWidth`). Поэтому работает И как полноэкранный слайдер (слайды `flex:0 0 100%` → шаг в ширину вьюпорта, как hero), И как многокарточный (узкие карточки с `gap` → шаг в карточку, видно «1.5 карточки», последний кадр прижимается к правому краю — см. блок «Это нужно знать»). Пересчёт на `resize`.
- **Центр-режим** `data-slider-center`: активный слайд центрируется во вьюпорте (`offsetLeft + width/2 - vp/2`), соседи выглядывают слева И справа (peek). Дополнительно `data-slider-start="N"` — стартовый индекс (чтобы изначально были видны оба соседа). Используется в блоке «О компании».

### Универсальный select (`partials/select.html` + `_select.scss` + `components/select.js`)
Кастомный селект (нативный `<select>` нельзя стилизовать как в макете). Подключение:

### Поток и логика (базовые редиректы)
- Регистрация: `login` → `register` → `verify` → `index`. Свитч-ссылки ведут назад (`register` → `login`, `verify` → `register`).
- Восстановление пароля: `login` (ссылка «Восстановить пароль») → `recover` → `recover-code` → `recover-password` → `login`. Свитч на всех — «Вспомнили пароль? / Войти в аккаунт».
- `recover-code`: код с `data-autosubmit` — при вводе 6-й цифры форма submit'ится автоматически (без кнопки подтверждения). Кнопка «отправить ещё раз» — `js-resend` + `data-resend-seconds`/`data-resend-label` (`resend-timer.js`): обратный отсчёт, по нулю становится активной, клик перезапускает таймер.
- Редирект формы — атрибутом `data-redirect="<page>"` на `<form>`; `js/components/form-redirect.js` перехватывает submit, прогоняет валидацию и переходит по адресу. Без бэкенда (заглушка под Битрикс).
- Валидация поля — `data-validate` на `.field`: `email` (формат) или `match` + `data-match="#id"` (совпадение, для повтора пароля). Ошибка показывается по `blur`, снимается при исправлении (`field-validate.js`).
- Код подтверждения `.code` (6 ячеек, `[data-code]`): авто-переход, Backspace на пустой → назад, стрелки, paste распределяет цифры (`code-input.js`). Заполненная ячейка — класс `is-filled`.
- Чекбокс согласия `.checkbox`: при попытке отправить без галочки — класс `is-invalid` (красная рамка), снимается по `change`. Галочка — иконка `icon-check`.

## Модалки и формы

### Модалка (`_modal.scss` + `components/modal.js`)
Универсальная модалка. Открытие — `data-modal-open="<id>"` на любом элементе (кнопка/ссылка), закрытие — `data-modal-close` (бэкдроп, крестик, кнопки) или `Esc`. Диалог `.modal__dialog` (radius 24, размеры `--sm`420/`--md`560/`--lg`760 через `--modal-width`), шапка `__header` (title+subtitle), `__body`, `__footer` (кнопки). При открытии — `html.modal-open` (блок скролла) + `lenis.stop()`. Скролл внутри длинного диалога — `data-lenis-prevent`, чтобы Lenis не перехватывал.
- **Мобилка:** футер `column-reverse` (главная кнопка сверху) + `.btn { flex: 0 0 auto }` — ОБЯЗАТЕЛЬНО, иначе `flex:1` в колонке схлопывает кнопки по высоте (flex-basis:0 → кнопки тонкие ~21px). Крестик `__close` на мобилке — серый круг (`$color-bg`, не белый: диалог почти на всю ширину, белый круг слился бы с белым фоном), позиционируется внутрь угла диалога.

### Модалка «Написать отзыв» (account-reviews.html, `#reviewFormModal`)
Открывается кнопкой «Написать отзыв» (`data-modal-open="reviewFormModal"`). Содержит форму `.review-form` (`_review-form.scss` — группы label+контрол, gap 24): **рейтинг** `.rating-input` (макс 5) + **textarea** `.review-form__textarea` (radius 16, как у селектов) + **аплоадер фото/видео** `.uploader`. Кнопки футера — заглушки (`data-modal-close`), под Битрикс.
- **Рейтинг-инпут `.rating-input`** (`_rating-input.scss` + `components/rating-input.js`) — 5 кликабельных звёзд (`icon-star`), значение в `<input type="hidden" data-rating-value>`. Ховер подсвечивает до наведённой, клик фиксирует, `mouseleave` возвращает к выбранному. Активные — золотые (`#f5b301`), неактивные — `rgba(black,.15)`. Это ВВОД (в отличие от display-`.rating` на карточках отзывов).
- **Аплоадер `.uploader`** переиспользован из модалки фото профиля (account-profile.html `#photoModal`), но `accept` расширен на видео (`mp4/webm`) и добавлен показ имени файла `.uploader__filename` (`[data-uploader-name]`). `uploader.js`: на любой файл вешает `has-file` + пишет имя; превью-картинку (кружок `__preview`) рисует только для image и только если элемент превью есть (в отзыве превью нет — просто имя файла акцентом).
- **Кнопка «Удалить» по статусу отзыва:** у `.review-card[data-status='published']` кнопка `.account__action--danger` скрыта (`display:none`) — опубликованный отзыв удалять нельзя, только редактировать; на `pending` (модерация) — обе кнопки. В вёрстке отражено через CSS по `data-status`; в Битриксе можно так же не рендерить кнопку серверно.

## Грабли (на которые уже наступали)

1. **Не писать литерал `@@include(...)` в HTML-комментариях** — gulp-file-include принимает за директиву.
2. **Автопрефиксер ломает `:placeholder-shown`** — добавляет `:not(:-moz-placeholder)` в ту же группу, Chrome её не знает → всё правило мёртвое. Поэтому скрытие лейбла поля — через JS-класс, а не `:placeholder-shown`.
3. **`opacity` на родителе обрезает детей** — для смешанного текста со ссылками использовать `rgba()` на цвете.
4. Предупреждение `legacy-js-api` от gulp-sass при сборке — безобидное.

## Деплой (Vercel)

- `vercel.json` уже настроен: `buildCommand: npm run build`, `outputDirectory: dist`.
- Репозиторий git инициализирован. Путь А: создать репо на GitHub → `git remote add origin ...` → `git push -u origin main` → импорт на [vercel.com/new](https://vercel.com/new). Framework — Other (сборка/папка подхватятся из `vercel.json`).
- Путь Б (без GitHub): `npm i -g vercel` → `vercel` → `vercel --prod`.

## Превью для разработки

`.claude/launch.json` содержит конфиги `dev` (npm run dev, порт 3000) и `preview` (лёгкий статический сервер на свободном порту для скриншотов).

## Страницы-заглушки

Чтобы ссылки навигации не вели в 404, для ещё не готовых разделов сделаны заглушки (хедер/футер + заголовок + «Раздел в разработке») через партиал `partials/stub.html` (`@@include('partials/stub.html', { "title": "..." })`, стиль `.stub`): `catalog`, `compare`, `favorites`, `cart`. Когда раздел верстается — содержимое `<main>` заполняется вместо `@@include` заглушки.
